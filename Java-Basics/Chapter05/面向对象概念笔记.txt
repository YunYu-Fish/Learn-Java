- 从语言方面讲：
  * 对于C语言来说，是完全面向过程的
  * 对于C++语言来说，是一半面向过程，一半面向对象
  * 对于Java语言来说，是完全面向对象的

- 什么是面向过程的开发方式？
  * 面向过程的开发方式的主要特点是：
    第一步干什么
    第二步干什么
    ...

  * 面向过程也注重实现功能的因果关系
    因为A所以B
    因为B所以C
    因为C所以D
    ...

  * 面向过程中没有对象的概念，只是实现这个功能的步骤以及因果关系

  * 面向过程有什么缺点？
    面向过程最主要是每一步与每一步的因果关系，其中A步骤因果关系到B步骤，A和B联合起来形成一个子模块，子模块和子模块直接又因为
    因果关系结合在一起，假设其中任何一个因果关系出现问题（错误），此时这个系统的运转都会出现问题（代码和代码直接的耦合度太高，
    扩展里太差）

  * 面向过程有那些优点？
    对于小型项目（功能），采用面向过程的方式进行开发，效率较高，不需要前期进行对象的提取，模型的建立，采用面向过程方式可以直接开始干活，一上来
    直接写代码，编写因果关系，从而实现功能

- 什么是面向对象
  * 采用面向对象的方式尽心开发，更符合人类的思维方式，更容易让我们接受
    人类就是以“对象”的方式去认识世界的

  * 面向对象就是讲现实世界分割成不同的单元，然后每一个单元都实现成对象，再驱动一下，让各个对象之间协作起来形成一个系统

  * 例：对象“张三” 对象“香烟” 对象“打火机” 对象“吸烟的场所”
       将以上四个对象组合在一起，就可以模拟一个人的抽烟场景
       其中“张三”对象可以更换为“李四”，“香烟”对象可以换品牌，“打火机”和“吸烟的场所”也可以更换

  * 采用面向对象的方式进行开发
    耦合度低，扩展力强

- 面向过程和面向对象关注的区别？
  面向过程主要关注的是：实现步骤以及整个过程
  面向对象主要关注的是：对象A，对象B，对象C..然后组合ABC或CBA...

- 当我们采用面向对象的方式贯穿整个系统的话，涉及到三个术语：
  OOA：面向对象分析
  OOD：面向对象设计
  OOP：面向对象编程
  整个软件开发的过程，都是采用OO进行贯穿的

- 面向对象包括三大特征
  封装
  继承
  多态
  任何一个面对对象的编程语言都包括这三个特征

- 类和对象的概念
  * 什么是类？
    类在现实中实际上是不存在的，是一个抽象的概念，是一个模版，是人类大脑进行思考、总结、抽象的一个结果

  * 什么是对象
    对象是实际存在的个体

  * 例：明星是一个类
       宋小宝就是一个对象
       姚明是一个对象
       刘德华是一个对象
       ...
       宋小宝、姚明、刘德华这3个对象都属于“明星”这个类

 - 在java语言中，要想得到“对象”，必须先定义“类”，“对象”是通过“类”这个模版创造出来的
   类就是一个模版：类中描述的是所有对象的“共同特征信息”
   对象就是通过类创建出的个体

- 关键术语
  类：不存在的，人类大脑思考总结一个模版（这个模版当中描述了共同特征）
  对象：实际存在的个体
  实例：对象的另一个名字
  实例化：通过类这个模版创建对象的过程，叫做实例化
  抽象：多个对象具有共同特征，进行思考总结抽取共同特征的过程


- 类的定义
  * 怎么定义一个类，语法格式是什么？
    （修饰符列表）class 类名{
        //类体 = 属性 + 方法
        //属性在代码上以“变量”的形式存在
        //方法描述动作/行为
    }
    ！！：修饰符列表可以省略

  * 为什么属性是以变量的形式存在的？
    因为属性对应的是数据，数据在程序当中只能放到变量中
